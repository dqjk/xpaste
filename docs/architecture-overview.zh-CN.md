# 架构概览

[English](./architecture-overview.md)

## 系统形态

`xpaste` 是一个中心化的浏览器/服务器应用：

- 一个服务进程运行在某台主机上。
- 其他设备通过浏览器打开网页。
- 服务端在内存中维护设备在线状态。
- 服务端通过 SSE 向客户端推送状态变化。
- 客户端通过 HTTP 发布文本、图片、视频和文件。

## 运行流程

1. 浏览器加载 `index.html`。
2. 客户端确保存在持久化的 `deviceId` Cookie。
3. 客户端建立 `GET /events` 连接。
4. 服务端注册连接并立即发送 `device-list`。
5. 客户端通过 JSON `POST /data` 发布文本。
6. 客户端通过 multipart `POST /data` 发布二进制内容。
7. 服务端广播 `data-created`、`device-connected` 和 `device-offline`。

## 存储模型

- 在线设备：内存
- SSE 连接：内存
- 每台设备的近期数据：内存
- 二进制内容：操作系统临时目录

服务端仅保留每台设备最近的 `N` 条数据，并在最后一个活跃会话断开后删除该设备状态。

## 分层规则

- `shared`：客户端与服务端共用的协议及类型契约
- `server`：HTTP、SSE、在线状态、数据生命周期和临时文件处理
- `client`：浏览器状态、渲染、上传操作和事件处理

业务逻辑与视图逻辑保持隔离。服务端路由不持有业务状态，客户端渲染器不负责数据请求。
